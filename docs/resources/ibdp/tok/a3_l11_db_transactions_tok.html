Transactions and Commit

A bank transfer example for IB Computer Science and TOK

Big idea
A transaction keeps a multi-step update together, and COMMIT makes it permanent.

The bank transfer

Alice
1000
Before transfer
Step 1: subtract 100
→
Bob
600
Before transfer
Step 2: add 100
SQL
START TRANSACTION;
Action
Two updates, one logical task
Finish
COMMIT; saves both

SQL example

START TRANSACTION;

UPDATE accounts
SET balance = balance - 100
WHERE id = 1;

UPDATE accounts
SET balance = balance + 100
WHERE id = 2;

COMMIT;
Atomicity
Both updates happen together, or neither happens.
Consistency
The data still follows the rules after the transfer.
Durability
After COMMIT, the result stays saved even after a crash.

What if the system crashes?

Without a transaction
  • • Alice loses 100.
  • • Bob does not receive 100.
  • • The database becomes inconsistent.
With transaction + rollback
  • • The failed transfer is undone.
  • • Alice keeps her original balance.
  • • The database remains trustworthy.
Key teaching line: a database should never show a “half-finished truth.”
